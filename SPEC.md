# SPEC.md — Техническое задание AI-Buyer

## Описание продукта

AI-Buyer — система автоматического контроля цен закупок с использованием AI. Снабженец загружает счёт от поставщика, система извлекает данные, ищет рыночные цены, сравнивает с историей закупок и формирует отчёт для ревизора/руководителя.

---

## Пользовательские роли

### Admin
- Полный доступ ко всему
- Управление пользователями и организацией
- Настройки системы, API-ключи

### Manager (Ревизор)
- Просмотр всех счетов и отчётов
- Аналитика и дашборд
- Настройка порогов и правил проверки
- Управление поставщиками и товарами

### Uploader (Снабженец)
- Загрузка счетов
- Просмотр своих загруженных счетов и результатов
- Внесение уточнений по позициям (если AI запрашивает)

---

## Функциональные требования

### FR-1: Загрузка счёта
- Пользователь загружает файл (PDF, DOCX, XLSX, JPG, PNG)
- Опция: «Только проверить» (без записи в историю) или «Проверить и записать»
- Опция: указать дополнительные расходы (доставка/упаковка, если не в счёте)
- Файл сохраняется в S3-хранилище
- Создаётся запись в БД со статусом `uploaded`
- Задача отправляется в очередь обработки

### FR-2: Парсинг счёта (AI)
- Определение формата документа
- OCR для изображений и сканов PDF
- Извлечение через LLM:
  - Поставщик (название, ИНН если есть)
  - Дата счёта, номер
  - Список позиций: наименование, количество, единица измерения, цена за единицу, сумма
  - НДС: включён в цену или сверху, ставка, сумма
  - Допрасходы: доставка, упаковка и т.д.
  - Итого
- Нормализация названий товаров
- Сопоставление поставщика с базой (или создание нового)
- Статус меняется на `parsed`

### FR-3: Проверка цен
- По каждой позиции:
  1. Поиск рыночной цены (веб-поиск с учётом объёма закупки)
  2. Поиск в истории закупок (своя база + pgvector для похожих товаров)
  3. Сравнение с ценами у других поставщиков
- Вычисление базовой единицы (если в счёте нестандартная фасовка)
- Оценка: `ok` (отклонение <10%), `attention` (10-25%), `overpriced` (>25%), `unknown` (нет данных)
- Статус меняется на `done`

### FR-4: Отчёт по счёту
- Таблица позиций с оценками
- По каждой позиции: цена в счёте, рыночная цена, цена в истории, % отклонения
- Общий вердикт по счёту
- Выявление допрасходов и их пропорциональности
- Выявление резкого изменения цен (если позиция ранее закупалась)

### FR-5: Уведомления
- Telegram: отправка краткого отчёта при обнаружении завышений
- Email: полный отчёт с деталями
- Настраиваемый порог (по умолчанию — уведомлять при отклонении >15%)
- Уведомление менеджеру (ревизору) при загрузке нового счёта

### FR-6: История и аналитика
- График динамики цен по товару
- Рейтинг поставщиков (средний уровень цен vs рынок)
- Сводка за период: сколько счетов, сколько завышений, потенциальная экономия
- Экспорт отчётов в PDF и XLSX

### FR-7: Справочники
- Товары: автопополнение при обработке, ручное редактирование, категории
- Поставщики: автопополнение, объединение юрлиц, ручное редактирование

---

## Нефункциональные требования

- **Время обработки счёта**: до 2 минут (5-10 позиций)
- **Одновременные пользователи**: до 50
- **Хранение**: минимум 2 года истории
- **Безопасность**: JWT, RBAC, HTTPS, шифрование файлов в хранилище
- **Отказоустойчивость**: retry для AI-вызовов (3 попытки), graceful degradation
- **Логирование**: все AI-вызовы логируются (prompt, response, tokens, cost)

## API Endpoints (Backend)

### Auth
```
POST   /api/auth/register      — Регистрация (только admin)
POST   /api/auth/login         — Вход
POST   /api/auth/refresh       — Обновление токена
POST   /api/auth/logout        — Выход
```

### Users
```
GET    /api/users              — Список (admin)
GET    /api/users/me           — Текущий пользователь
PATCH  /api/users/:id          — Обновление (admin)
DELETE /api/users/:id          — Удаление (admin)
```

### Invoices
```
POST   /api/invoices           — Загрузка счёта (multipart/form-data)
GET    /api/invoices            — Список (с фильтрами и пагинацией)
GET    /api/invoices/:id       — Детали счёта с позициями
GET    /api/invoices/:id/report — Отчёт по счёту
DELETE /api/invoices/:id       — Удаление (admin/manager)
POST   /api/invoices/:id/recheck — Повторная проверка
```

### Suppliers
```
GET    /api/suppliers          — Список
GET    /api/suppliers/:id      — Детали + статистика
PATCH  /api/suppliers/:id      — Обновление
POST   /api/suppliers/:id/merge — Объединить юрлица
```

### Products
```
GET    /api/products           — Список с поиском
GET    /api/products/:id       — Детали + история цен
PATCH  /api/products/:id       — Обновление
GET    /api/products/:id/price-history — История цен
```

### Analytics
```
GET    /api/analytics/summary      — Сводка за период
GET    /api/analytics/savings      — Потенциальная экономия
GET    /api/analytics/suppliers    — Рейтинг поставщиков
GET    /api/analytics/price-trends — Тренды цен
```

### Reports
```
GET    /api/reports                — Список отчётов
GET    /api/reports/:id/export/pdf — Экспорт в PDF
GET    /api/reports/:id/export/xlsx — Экспорт в XLSX
```

## Промпты для AI Service

### Извлечение данных из счёта

```
Ты — AI-ассистент для анализа закупочных документов.

Проанализируй следующий документ (счёт на оплату / коммерческое предложение) и извлеки:

1. Поставщик: название, ИНН (если указан)
2. Номер документа, дата
3. Список позиций. По каждой:
   - Наименование товара/услуги (как указано в документе)
   - Количество
   - Единица измерения
   - Цена за единицу
   - Сумма
4. НДС: включён в стоимость или сверху, ставка (%), сумма
5. Дополнительные расходы: доставка, упаковка, и т.д.
6. Итоговая сумма

Верни результат СТРОГО в JSON формате:
{
  "supplier": {"name": "...", "inn": "..."},
  "document": {"number": "...", "date": "YYYY-MM-DD"},
  "items": [
    {"name": "...", "quantity": 0, "unit": "...", "price_per_unit": 0, "total": 0}
  ],
  "vat": {"included": true/false, "rate": 20, "amount": 0},
  "extra_costs": [{"type": "delivery", "amount": 0}],
  "total": 0,
  "currency": "RUB",
  "confidence": 0.95
}

Если не можешь определить значение — ставь null.
```

### Оценка цены позиции

```
Ты — AI-аналитик закупок.

Данные:
- Товар: {name}
- Цена в счёте: {price} руб. за {unit} (количество: {qty})
- Рыночная цена (поиск): {market_price} руб. (источник: {source})
- Средняя цена в истории закупок: {history_avg} руб. (за последние {period})
- Цена при прошлой закупке у этого поставщика: {last_price} руб.

Оцени:
1. Отклонение от рыночной цены (%)
2. Отклонение от средней по истории (%)
3. Изменение по сравнению с прошлой закупкой у этого поставщика (%)
4. Учёт объёма закупки (оптовая скидка)
5. Итоговая оценка: ok / attention / overpriced / unknown
6. Краткое пояснение

Верни JSON:
{
  "assessment": "ok|attention|overpriced|unknown",
  "market_deviation_pct": 0,
  "history_deviation_pct": 0,
  "supplier_change_pct": 0,
  "explanation": "..."
}
```
